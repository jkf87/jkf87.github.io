![오픈클로 2026.5.28 업데이트 히어로 이미지](/images/2026-06-01-openclaw-528-update/hero.jpg)

5.27이 보안과 안정성의 기둥을 세웠다면, 5.28은 그 위에 올라탄 실전 개선의 층이다. 에이전트가 죽지 않고 버티는 법, 채널이 메시지를 잃지 않는 법, 모바일이 끊김 없이 연결되는 법 — 겉보기엔 조용한 업데이트지만, 매일 에이전트를 돌리는 사람에게는 체감이 다른 버전이다.

## 에이전트가 안 죽는다 — 런타임 회복력 대폭 강화

이번 업데이트의 뼈대는 "회복력(resilience)"이다. 서브에이전트가 작업 디렉토리와 워크스페이스를 헷갈리지 않고, 세션 락이 타임아웃에 걸려도 정상적으로 해제되며, Codex 앱 서버나 헬퍼가 죽어도 공유 런타임 상태가 무너지지 않는다.

이전까지는 Codex 런타임에서 헬퍼가 실패하면 공유 상태가 오염되어 복구가 안 됐다. 5.28에서는 이 격리가 제대로 작동한다. 스테일한 재시작 연속성(stale restart continuation)도 방지해서, 재시작 후 이전 세션의 잔해가 새 세션을 꼬이게 하지 않는다.

> **한 줄 요약:** 서브에이전트·Codex·세션 락 — 어디서 문제가 나든 연쇄 장애로 번지지 않는다.

## Claude Opus 4.8, Fal Krea, NVIDIA — 모델 지원이 또 넓어졌다

새로운 모델이 줄줄이 들어왔다:

- **Claude Opus 4.8** — Anthropic 최신 모델 지원
- **Fal Krea** — 이미지 생성 모델 스키마 추가
- **NVIDIA Featured Models** — NVIDIA 카탈로그 연동
- **MiniMax 스트리밍 음악** — 음악 생성 응답 스트리밍 지원
- **암호화 PDF 추출** — ClawPDF로 암호 걸린 PDF도 읽는다
- **음성 모델 카탈로그** — 프로바이더 기반 음성 모델 목록 지원

GitHub Copilot 에이전트 런타임도 추가되고, Codex Supervisor 플러그인 패키지로 위임형 Codex 워크플로우가 가능해졌다. 이건 Codex를 직접 쓰지 않고도 Codex 능력을 빌려 쓰는 길이 열렸다는 뜻이다.

## 채널이 메시지를 잃지 않는다

5.27에서 시작된 채널 안정화가 5.28에서 마저 완성됐다. Telegram 폴링이 끊기지 않고, iMessage 리액션/승인이 중복 없이 작동하며, Slack이 마지막 응답을 잃지 않고, Discord가 복구된 도구 경고를 깔끔하게 처리한다.

Microsoft Teams 서비스 URL 신뢰 검사도 강화됐고, WhatsApp 프로필 인증 루트가 제자리를 찾았다. Matrix 방 ID 대소문자 처리도 수정됐다 — 이런 디테일이 쌓여서 "어느 순간 메시지가 안 간다"는 문제가 사라진다.

## 모바일 — iOS Pro UI가 본격적으로 돌아간다

iOS 개발 앱이 Pro Command, Chat, Agents, Settings를 갖췄다. 호스팅된 푸시 릴레이 기본값이 설정되고, 실시간 Talk 탭 재생이 게이트웨이 세션에 연결됐다. WebChat 재연결 시 상태 유지도 개선됐고, 세션 피커가 빈 검색에서도 정상 동작한다.

> 모바일에서 에이전트를 쓰는 경험이 데스크톱에 한 걸음 더 가까워졌다.

## 보이지 않는 곳의 견고함 — 파싱·설정·보안

사용자 눈에 띄지 않지만 중요한 변화가 많다:

- **입력 검증 강화** — 잘못된 탭 인덱스, 과도한 뷰포트 리사이즈, 0번 CDP 포트, 변형된 지오테이션 옵션 등을 조기에 거부
- **설정 파싱** — 숫자·타임아웃·버전 입력 검증, IPv6 프록시 지원, 비정상 미디어 콘텐츠 길이 거부
- **Cron 자동화** — 일시적 모델 속도 제한 후 재시도, 예약 작업 스킵 전 모델 폴백 사전 점검
- **플러그인 핫패스 최적화** — 도구 검색 카탈로그, 세션 스토어, 매니페스트 모델 행 캐시가 재계산을 줄인다

Diffs 언어 지원도 확장됐고, ClawHub에 플러그인 표시 이름과 스킬 검증/신뢰 서피스가 추가됐다. 워크보드에 에이전트 조정 도구도 들어왔다 — 활성 에이전트 작업을 추적하고 인계하는 용도다.

## 5.27과 5.28 — 연속된 두 발

5.27이 보안 경계와 채널 기반을 다졌다면, 5.28은 그 기반 위에서 런타임 회복력과 모델 확장을 채웠다. 혼자 쓰는 에이전트든, 팀 전체가 쓰는 멀티에이전트 시스템이든 — "갑자기 안 되는" 순간이 줄어드는 게 이번 업데이트의 핵심이다.

그리고 그 "갑자기 안 되는" 순간이 줄어든다는 건, 에이전트를 프로덕션 도구로 믿을 수 있게 된다는 뜻이다.
