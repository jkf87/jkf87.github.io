LLM 기반 에이전트가 매번 새로운 문제를 풀 때마다 처음부터 시작한다면? 비효율적이겠죠. Google Cloud AI Research와 UIUC, MIT가 함께 발표한 SkillOS는 에이전트가 과거 경험에서 "스킬"을 추출하고, 이를 정제·관리·재사용하는 능력을 강화학습으로 학습하는 새로운 프레임워크입니다.

이 글에서는 arXiv 2605.06614에 게재된 "SkillOS: Learning Skill Curation for Self-Evolving Agents" 논문의 핵심 내용을 정리합니다.

📌 원문: https://arxiv.org/abs/2605.06614


왜 "스킬 큐레이션"이 중요한가?

LLM 에이전트가 스트리밍 환경에서 연속적으로任务를 처리할 때, 이전 경험을 재사용하는 능력, 즉 "자기 진화(self-evolution)"가 필수적입니다. 이를 위한 핵심 기반은 절차적 기억(procedural memory) — 특히 "재사용 가능한 스킬"입니다.

하지만 기존 접근법들은 한계가 뚜렷합니다:

• 수동 큐레이션 (예: Anthropic의 스킬 저장소): 전문 지식이 많이 필요하고, 에이전트가 만날 수 있는 다양한 태스크에 대응 불가
• 휴리스틱 기반 메모리 관리: 고정된 규칙에 의존하고, 하위 성능 피드백이 없어 실제 에이전트 필요에 맞추지 못함
• 기존 RL 접근법: 스킬 "사용"을 가르치거나 짧은 태스크 스트림 내에서만 최적화 — 복잡한 장기 관리 연산(update, delete)에 대한 학습 신호 부족


SkillOS의 핵심 아이디어

SkillOS는 "스킬 큐레이션"을 장기적이고 실행기 기반의 학습 문제로 정식화합니다.

멀티 에이전트 모듈형 설계

시스템은 두 가지 역할로 분리됩니다:

① Agent Executor (동결된 실행기)
태스크를 받으면 현재 SkillRepo에서 관련 스킬을 검색(BM25)하여 태스크를 해결합니다. 실행기는 고정(frozen)되어 있으며 스킬을 소비하기만 합니다.

② Skill Curator (학습 가능한 큐레이터)
실행기가 태스크를 마친 후, 궤적(trajectory)과 결과를 관찰하고 SkillRepo에 세 가지 연산을 수행합니다:
• insert_skill — 새로운 스킬 추가
• update_skill — 기존 스킬 수정
• delete_skill — 불필요/유해 스킬 제거

스킬은 SKILL.md 형식의 Markdown 파일로 표현됩니다. YAML 프론트매터(이름, 사용 시점 설명)와 Markdown 지침(실행 가능한 지식, 워크플로, 제약조건)으로 구성됩니다.


RL 훈련 레시피의 두 가지 핵심 설계

1) 관련 태스크 그룹핑
각 훈련 인스턴스를 "관련 태스크 그룹"으로 구성합니다. 앞선 태스크들의 궤적이 SkillRepo를 업데이트하고, 뒤에 오는 관련 태스크들이 이 업데이트를 평가합니다. 이를 통해 장기적 유용성에 기반한 큐레이션 학습이 가능해집니다.

2) 복합 보상(Composite Reward)
지연되고 간접적인 피드백을 학습 신호로 변환하기 위해 여러 보상을 조합합니다:
• 태스크 성과 (downstream executor feedback)
• 유효한 함수 호출 여부
• 스킬 품질
• SkillRepo의 간결성(compactness)


실험 결과

멀티턴 에이전트 태스크와 싱글턴 추론 태스크 모두에서 SkillOS가 일관되게 우수한 성능을 보였습니다.

성능 개선

• 최대 +9.8% 상대 성능 향상
• 최대 -6.0% 상호작용 단계 감소
• 메모리 없는 기준선과 강력한 메모리 기반 기준선 모두를 능가

일반화 능력

• 학습된 큐레이터는 다른 실행기 백본과 태스크 도메인에도 잘 일반화됨
• Gemini-2.5-Pro를 실행기로 사용할 때도 성능 향상
• 주목할 만한 점: 8B 큐레이터 모델이 Gemini-2.5-Pro를 직접 큐레이터로 사용할 때보다 더 나은 성능 달성


정성 분석

• 학습된 큐레이터는 더 타겟팅되고 효과적인 스킬 활용을 유도
• SkillRepo의 스킬들이 시간이 지나면서 더 풍부한 구조를 가진 Markdown 파일로 진화
• 점차 더 높은 수준의 메타 스킬(meta-skills)이 등장하는 것을 관찰


개인적인 생각

이 논문이 흥미로운 이유는 "에이전트가 자신의 경험을 어떻게 정리할 것인가"라는 근본적 질문에 답하기 때문입니다. 기존에는 에이전트의 메모리 관리를 규칙이나 프롬프트로 해결하려 했지만, SkillOS는 이 자체를 학습 문제로 접근합니다.

특히 인상적인 부분:

• "스킬 사용"이 아니라 "스킬 관리"를 학습 대상으로 삼은 것 — 실제 배포 환경에서 오래된 스킬을 갱신하고 해로운 스킬을 삭제하는 능력은 확장성에 필수적
• 8B 모델이 Gemini-2.5-Pro보다 큐레이터로서 더 나은 성능을 보인 것 — 큐레이션 능력이 모델 크기에 비례하지 않으며, 전문 학습이 중요하다는 증거
• SkillRepo가 시간이 지나면서 메타 스킬을 발달시킨다는 관찰 — 자기 진화의 실질적 증거

다만 몇 가지 질문도 남습니다:

• BM25 기반 검색은 간단하지만, 의미적 유사도 기반 검색으로 교체하면 어떨까?
• 스킬이 Markdown 파일이라 토큰 소모가 큰데, 더 압축된 표현은 가능할까?
• 실제 배포 환경에서 SkillRepo의 크기가 무한히 커지는 것을 어떻게 제어할 것인가?
