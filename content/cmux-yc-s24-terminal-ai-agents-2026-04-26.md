![수십 개 에이전트 동시 운영하는 카오스 — 그게 cmux가 풀려는 문제임](./images/cmux-yc-s24-terminal-ai-agents-2026-04-26/multitasking.gif)

1\. 새 터미널이 떴음. 이름은 **cmux**.

2\. 만든 사람은 **Austin Wang**. YC S24 출신 창업자임. 회사 이름은 **manaflow-ai**.

3\. 처음엔 그냥 또 다른 터미널인가 했음. 근데 콘셉트가 다름.

4\. **Cursor도 VS Code도 버리고 만들었다**고 함. 이유가 핵심임.

5\. 진짜 문제는 코드 작성이 아니라 **수십~수백 개 에이전트 동시 운영**이었음.

6\. Claude Code, Codex 같은 거 5개만 띄워도 문제가 시작됨. 어느 놈이 멈췄는지 모름.

7\. tmux 쓰면 분할은 되는데 알림이 통합 안 됨. macOS 기본 알림은 컨텍스트가 비어있음.

8\. 그래서 cmux가 들고 온 답이 **알림 링(Notification Rings)**.

![에이전트 100개 띄워놨을 때 알림이 안 울리면 이 짤대로 됨](./images/cmux-yc-s24-terminal-ai-agents-2026-04-26/notification-ring.gif)

9\. 에이전트가 입력 기다리면 그 패널에 **파란 링**이 들어옴. 탭에도 동일하게 표시.

10\. 단축키 `⌘⇧U`로 가장 최근 미해결 알림으로 바로 점프함. 패널 100개 띄워놔도 동선이 안 끊김.

11\. 두 번째 무기는 **사이드바**. 일반 터미널엔 없는 정보가 박혀있음.

![터미널 띄워놓고 사이드바에 git/PR/포트까지 보면 사실상 이 짤 됨](./images/cmux-yc-s24-terminal-ai-agents-2026-04-26/hacker-terminal.gif)

12\. git 브랜치, PR 상태, 작업 디렉토리, 열린 포트, 알림 컨텍스트가 워크스페이스마다 표시됨.

13\. 즉 "지금 이 에이전트가 어느 브랜치에서 무슨 작업 중인지" 한눈에 보임.

14\. 세 번째가 **내장 브라우저**. 단순 임베드가 아님.

15\. **스크립팅 API**가 붙어있음. 접근성 트리 스냅샷, 엘리먼트 클릭, 폼 자동 채움까지 됨.

16\. 즉 에이전트가 자기 옆에서 직접 브라우저 띄우고 테스트 돌릴 수 있음. 맥락 안 깨짐.

17\. 네 번째는 **SSH 워크스페이스**. 원격 서버에서 작업할 때 그 네트워크 안 브라우저까지 라우팅함.

18\. 다섯 번째 — **Claude Code Teams** 네이티브 통합. 팀원이 띄운 에이전트랑 실시간 협업되는 모드.

![Claude Code Teams 모드 — 팀원 에이전트랑 같이 끌고 가는 그림](./images/cmux-yc-s24-terminal-ai-agents-2026-04-26/teamwork.gif)

19\. 여섯 번째 — **세션 복원**. 레이아웃, 디렉토리, 스크롤백, 브라우저 히스토리까지 다 보존됨.

20\. 일곱 번째 — `cmux.json` 파일로 프로젝트별 커스텀 커맨드 등록 가능.

21\. 기술적으로 보면 Electron 안 씀. **Swift + AppKit 네이티브**임.

22\. 렌더링은 **libghostty** 위에 올렸음. GPU 가속 터미널 그대로 가져옴.

23\. 그래서 `~/.config/ghostty/config` 그대로 읽음. Ghostty 쓰던 사람은 테마/설정 무손실 이전됨.

24\. 설치는 두 가지. DMG 직접 다운, 아니면 Homebrew로 `brew tap manaflow-ai/cmux && brew install --cask cmux`.

25\. 가격은 **무료**임. **GPL-3.0** 오픈소스.

26\. 유료 옵션도 있음. **Founder's Edition** — 우선 지원 + 얼리 피처 액세스. 강제 아님.

27\. 여기까지가 기능. 이제 의미를 봐야 함.

28\. cmux는 단순한 터미널 교체가 아님. **AI 에이전트 운영 OS 레이어**임.

29\. 정확히는 Cursor가 IDE를 갈아엎었듯, cmux는 터미널을 갈아엎으려는 시도임.

30\. 근데 진짜 통찰은 단순함. **"에이전트가 멈췄을 때 빨리 돌아가는 것"** 하나에 모든 기능이 정렬됨.

31\. 알림 링, 사이드바 메타, 단축키, 세션 복원 — 전부 "흐름 안 끊기게"가 목적.

32\. 한국 개발자 입장에서는 어떻게 봐야 하나.

33\. 지금은 1명당 에이전트 1~2개 돌리는 단계임. 그러면 cmux 굳이 안 필요함.

34\. 근데 1명이 5개+ 동시에 돌리는 단계 가면 얘기가 달라짐. tmux + 메모만으로는 안 굴러감.

35\. 한국에서도 Claude Code, Codex 쓰는 개발자 빠르게 늘고 있음. 6~12개월 후 5개+ 운영이 표준 될 가능성 큼.

36\. 그때 cmux 같은 도구가 표준 인프라가 될 수 있음.

37\. 한계도 명확함. **macOS 전용**. 윈도우, 리눅스 미지원. 한국 회사 환경에서는 일단 컷.

38\. 그래도 신호는 분명함. 도구 스택이 한 단 더 올라가고 있음.

39\. **모델 → 에이전트 → 에이전트 오케스트레이션 도구**. 다음 단계는 "사람 1명이 100명분 일하기".

40\. cmux는 그 흐름의 인프라 후보 중 하나임. 지금 깔리고 있는 거.
